# Human–agent collaboration — how Bob compounds this project

**Audience:** Bob (Laidlaw Scholar) and anyone briefing a new agent.  
**Companion:** [`AGENTS.md`](../AGENTS.md) (machine-facing rules).

---

## 1. The real scarce resource

Code is cheap relative to **shared, durable understanding**. Chat is a scratchpad; the repository is the compound-interest account.

Your job as scholar is not only to request outputs. It is to:

1. **Deposit context** after meetings and reading,
2. **Name the mode** (Ship / Explore / Teach / Decide),
3. **Close human-only gates** (data, governance, taste),
4. **Notice yourself becoming a better scientist** — questions getting sharper, not just slides getting prettier.

---

## 2. What you can do for agents (high leverage)

| Move | Why it compounds |
|---|---|
| After every lab meeting, write or ask for a **debrief note** (before → after → out of scope → next executable step) | Future agents inherit the meeting; they do not need you to re-explain |
| Point to files (`reports/...`, ledger IDs) instead of retyping lore | Reduces drift and contradiction |
| Say **done when** (audience, length, decision needed, forbidden claims) | Turns “highest quality” into a finish line |
| Name **Ship vs Explore** | Legitimises productive wandering without fake findings |
| Deliver **partial dictionaries** early (even ugly) | Unblocks schema/QC; prevents invented field meanings |
| Correct scientific framing early when something feels off | One sentence from you beats an hour of elegant wrongness |
| Keep Gate owners human (Bishai / Roro / you) | Agents should not role-play PI authority |

You are not an obstacle when you are slow. You are an obstacle only when context dies in your head.

---

## 3. Lab meetings as context engines

A meeting is successful for this repo if it changes at least one durable object:

- README “what this project is (now)”,
- assumption ledger row,
- decision gate status,
- pathway enabled/disabled,
- a new `reports/meeting_debrief_*.md`.

Slides persuade the room. **Debriefs persuade the future.**

Prefer: *report to the group* **and** *deposit into the ledger*.

---

## 4. Permission to stray (on purpose)

Accomplished scientists follow side paths, sit with confusion, and sometimes discover the real question only after a detour.

Make that explicit:

> “Explore for a while: chase whether monthly 2D3N is too sparse; write dead ends into the compound log. Do not promote anything to headline without Gate 3.”

That is not indecision. That is **research hygiene with a notebook**.

---

## 5. How you grow as a Laidlaw Scholar here

Growth markers (more important than polish):

1. You can explain the estimand in one paragraph without AMI slippage.
2. You can say what would **falsify** your preferred story.
3. You can tell a synthetic result from a real one under time pressure.
4. You ask for dictionaries and governance before asking for coefficients.
5. You leave meetings with written before/after, not only impressions.

Agents can accelerate craft. Only you can practice judgment.
